# 到期提醒（ExpiryReminder）— 需求与技术文档

## 一、项目概述

**应用名称**：到期提醒
**包名**：com.expiryreminder.app
**最低版本**：Android 7.0（API 24）
**目标版本**：Android 14（API 34）
**开发语言**：Kotlin
**UI 框架**：Jetpack Compose + Material 3

到期提醒是一款帮助用户管理各类物品有效期的 Android 应用。用户可以录入食品、证件、会员、药品等物品的到期时间，应用会自动计算剩余天数并在临期时发送通知提醒，避免物品过期浪费。

---

## 二、功能需求

### 2.1 首页

- 展示所有物品列表，按到期时间升序排列
- 顶部显示统计卡片：总物品数、即将到期数（7天内）、已过期数
- 支持搜索功能，按物品名称或备注模糊匹配
- 点击物品进入详情页
- 下拉刷新数据

### 2.2 添加物品

- **分类入口**：按 12 个预设分类（食品厨房、个人护理、健康医疗、证件文件、金融保险、会员订阅、数码设备、家居维护、宠物用品、出行旅游、工作学习、其他）进入
- **模板快速添加**：每个分类下提供常用物品模板（如牛奶、酸奶、肉类等），选择模板后自动填充名称和默认保质期
- **自定义添加**：手动输入物品名称、分类、到期日期、购买日期、数量、单位、存放位置、备注、提醒天数
- **图片拍摄/选择**：支持拍照或从相册选择物品图片，自动压缩为 WebP 格式存储
- **提醒设置**：可设置提前几天提醒（如提前3天、7天等）

### 2.3 物品详情

- 展示物品完整信息：名称、图片、分类、到期日期、剩余天数、购买日期、数量、位置、备注
- 支持编辑物品信息
- 支持删除物品

### 2.4 提醒物品

- 展示所有设置了提醒的物品
- 按到期时间排序
- 已过期物品不显示

### 2.5 我的页面

- 显示用户头像、昵称、签名
- 统计概览：总物品数、即将到期、已过期、安全物品数
- 功能入口：
  - 分类管理
  - 数据统计
  - 数据备份与同步（坚果云 WebDAV）
  - 个人资料编辑
  - 清除数据

### 2.6 分类管理

- 查看、新增、编辑、删除分类
- 支持二级分类结构（父分类 → 子分类）
- 12 个预设父分类，每个包含多个子分类和模板物品

### 2.7 数据统计

- 按分类统计物品数量
- 按到期状态统计（已过期、即将到期、安全）
- 环形图可视化展示

### 2.8 数据备份与同步

- **坚果云 WebDAV 同步**：通过 WebDAV 协议与坚果云进行数据同步
- **备份**：将物品数据（含图片）上传到坚果云 `ExpiryReminder/` 目录
  - 物品数据序列化为 JSON 上传至 `ExpiryReminder/backup.json`
  - 物品图片以 `{itemId}.webp` 命名上传至 `ExpiryReminder/images/` 目录
  - 物品与图片通过 itemId 一一对应
- **恢复**：从坚果云下载备份数据和图片到本地
  - 先下载 `backup.json` 解析物品和分类数据
  - 再根据 `hasImage` 标记逐个下载图片到本地 `filesDir/images/{itemId}.webp`
  - 恢复时保留物品原始 ID，确保图片与物品对应关系正确
- **验证**：通过 PROPFIND 请求验证坚果云账号和应用密码

### 2.9 桌面小组件

提供三种尺寸的桌面小组件：

| 小组件 | 尺寸 | 功能 |
|--------|------|------|
| 到期提醒统计 | 1×1 | 显示即将到期物品数量，点击打开 App |
| 到期提醒概览 | 4×1 | 显示最近到期物品列表和统计数据 |
| 到期提醒列表 | 4×2 | 左侧分类标签可点击筛选，右侧显示对应分类物品列表 |

- 所有小组件每 15 分钟自动刷新数据
- 已过期物品不在小组件中显示
- 小组件点击可跳转到 App 对应页面

### 2.10 通知提醒

- 每小时检查一次即将到期物品（7天内）
- 发送系统通知，采用 InboxStyle 展示多个物品
- 点击通知跳转到 App
- 通知渠道：到期提醒（高优先级）

## 六、关键业务逻辑

### 6.1 到期天数计算

统一使用当天零点作为比较基准，因为 `expireDate` 存储的是当天零点时间戳：

```kotlin
fun getDaysUntilExpiry(expireTimestamp: Long): Long {
    val calNow = Calendar.getInstance()
    val calExpire = Calendar.getInstance().apply { timeInMillis = expireTimestamp }
    // 统一重置到零点比较
    for (cal in listOf(calNow, calExpire)) {
        cal.set(Calendar.HOUR_OF_DAY, 0)
        cal.set(Calendar.MINUTE, 0)
        cal.set(Calendar.SECOND, 0)
        cal.set(Calendar.MILLISECOND, 0)
    }
    return TimeUnit.MILLISECONDS.toDays(calExpire.timeInMillis - calNow.timeInMillis)
}
```

### 6.2 图片处理流程

1. 用户拍照或选择图片 → 获取 Uri
2. `ImageCompressor.compressImage()` 压缩图片：
   - 目标尺寸：1280×960
   - 输出格式：WebP，质量 75
   - 存储路径：`filesDir/images/{timestamp}.webp`
3. 将压缩后的本地绝对路径存入 `item.imageUri`

### 6.3 坚果云同步流程

**备份（上传）**：
1. 创建 `ExpiryReminder/` 和 `ExpiryReminder/images/` 目录
2. 遍历物品，将有图片的物品图片文件以 `{itemId}.webp` 命名上传到 `images/` 目录
3. 序列化物品和分类数据为 JSON，`imageUri` 保存远程路径，新增 `hasImage` 标记
4. 上传 `backup.json`

**恢复（下载）**：
1. 下载 `backup.json` 解析数据
2. 对标记 `hasImage=true` 的物品，从 `ExpiryReminder/images/{itemId}.webp` 下载图片
3. 保存到本地 `filesDir/images/{itemId}.webp`，更新 `imageUri` 为本地路径
4. 恢复数据时保留物品原始 ID，确保图片与物品对应

### 6.4 小组件刷新机制

- WorkManager 每 15 分钟触发 `WidgetUpdateWorker`
- `WidgetUpdateHelper.updateAllWidgets()` 通知所有小组件更新
- 4×2 小组件支持分类切换：通过 `ActionCallback` 处理点击事件，`updateAppWidgetState` 更新状态
